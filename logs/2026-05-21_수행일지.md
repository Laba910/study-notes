## 4. LeRobot 데이터셋 변환 완료 ✅

MCAP → LeRobot v0.4.4 포맷으로 변환:

```bash
cd /home/laba/datasets/wrench_handover
source /opt/ros/jazzy/setup.bash
python3 convert_to_lerobot.py
```

**변환 결과:**
- 에피소드: 55개
- 프레임: **15,567 frames** (10fps 동기화)
- 파일 크기: 8.4GB (MCAP) → **64MB** (MP4 AV1)
- 출력 경로: `/home/laba/datasets/lerobot_dataset/wrench_handover/`
- 원본 raw_bags: **보존** (삭제하지 않음)

**데이터셋 구성:**
| 키 | 설명 |
|----|------|
| `observation.state` | pair1/follower 관절 6개 (float32) |
| `action` | pair1/leader 관절 6개 (float32) |
| `observation.images.cam_top_bird` | 224×224 버드아이뷰 영상 |
| `observation.images.pair1_cam` | 224×224 손목 카메라 영상 |

### 주요 구현 사항

- **타임스탬프 동기화**: 10fps 기준으로 각 프레임마다 가장 가까운 이미지/관절 상태 binary search
- **action 정의**: 리더(사람 시연)의 관절 위치값 = 학습 타겟
- **영상 인코딩**: LeRobot 내장 AV1/SVT-AV1 인코더 사용
- **환경 이슈**: `rosbag2_py`가 ROS2 시스템 Python에서만 동작 → 모든 LeRobot 의존성을 `pip3 install --break-system-packages`로 시스템에 설치

## Q&A 정리 (오늘 배운 것)

**Q. 그리퍼 정보가 없으면 잡았는지 어떻게 알아?**  
A. gripper_joint_1이 JOINT_ORDER에 포함돼 있어 그리퍼 개폐 상태가 state/action에 함께 기록됨.

**Q. 카메라 영상을 어떤 형식으로 저장하면 좋은가? (MP4 vs PNG)**  
A. LeRobot은 MP4(AV1 코덱)를 기본 사용. PNG 시퀀스 대비 용량 1/100 수준으로 줄어들면서 학습 속도도 빠름. AV1은 lossy지만 관절제어 학습에서는 영향 미미.

**Q. 나중에 다른 도구(망치 등) 데이터와 어떻게 합치나?**  
A. 도구별로 별도 폴더(`wrench_handover/`, `hammer_handover/`)에 수집 → 각각 변환 시 `task` 레이블 부여 → LeRobot concatenate API로 멀티태스크 데이터셋 병합.

**Q. 왜 fps=10인가?**  
A. pair1/cam 카메라의 하드웨어 최대 fps가 ~10fps. 가장 느린 센서에 맞춰 동기화해야 보간 없이 실제 측정값 유지.
