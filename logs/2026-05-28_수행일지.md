## 2. ROS2 제어 구조 파악 ✅

실제 배선을 디버깅하면서 파악한 pair1 제어 구조:

```
policy (run_policy_v2.py)
    ↓ /pair1/leader/joint_trajectory
arm_controller (JointTrajectoryController)   ← follower에 탑재, 이 토픽 직접 구독(remapped)
    ↓
follower 모터
```

**주의**: `joint_trajectory_command_broadcaster`가 리더 현재 위치를 100Hz로 같은 토픽에 발행 → policy 10Hz 명령과 충돌. policy 시작 시 반드시 deactivate 필요.

## 3. 디버깅 이력

### 버그 1: pair1/cam 채널 오류 (크래시)
- **원인**: usb_cam이 `yuv422_yuy2` (2채널)로 발행 → 모델이 3채널 기대해서 `RuntimeError: size mismatch`
- **해결**: `decode_image()` 함수에 YUV422 → RGB 변환 분기 추가
```python
elif enc in ('yuv422_yuy2', 'yuyv', 'yuv422', 'YUV422'):
    return cv2.cvtColor(arr.reshape(msg.height, msg.width, 2), cv2.COLOR_YUV2RGB_YUYV)
```

### 버그 2: broadcaster 충돌 (팔이 들썩임)
- **원인**: `joint_trajectory_command_broadcaster`가 100Hz로 리더 현재 위치 덮어씀 → policy 10Hz 명령 묻힘
- **해결**: policy 시작 시 broadcaster 자동 deactivate
```python
subprocess.run(['ros2', 'control', 'switch_controllers',
    '--controller-manager', '/pair1/leader/controller_manager',
    '--deactivate', 'joint_trajectory_command_broadcaster'], ...)
```

### 버그 3: 잘못된 토픽 (명령 무시됨)
- **원인**: `arm_controller`는 `/pair1/leader/joint_trajectory` 구독(remapped)인데 `/pair1/follower/arm_controller/joint_trajectory`에 발행
- **해결**: `TRAJ_TOPIC = '/pair1/leader/joint_trajectory'`

### 버그 4: 그리퍼 부호 반전 (닫힌 채로 일어남)
- **발견 경위**: 학습 데이터 분석 결과, 리더암과 팔로워암의 그리퍼 부호 규칙이 반대임 확인
  - 리더 열림 = `-0.67`, 팔로워 열림 = `+0.67`
  - 학습 action의 gripper=-0.672는 "리더 열림" = 팔로워에 그대로 보내면 "팔로워 닫힘"
- **해결**: `_publish_action()`에서 그리퍼만 부호 반전
```python
positions[0] = -positions[0]  # gripper_joint_1 부호 반전
```

### 버그 5: 시작 자세 불일치 (엉뚱한 방향 이동)
- **원인**: `standby_human` (j2=-2.106) ≠ 학습 시작 자세 (j2=-1.843) → distribution shift
- **해결**: `modes_config.json`에 `policy_start` 모드 추가 (55 에피소드 평균값)
```json
"policy_start": [-0.670, -0.007, -1.843, 1.633, 1.409, -0.035]
```
